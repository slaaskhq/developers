Functions

Function Description
_slaask.show() Show the widget and trigger slaask.open event if hidden
_slaask.hide() Hide the widget and trigger slaask.close event if hidden
_slaask.message(message, from, { 'text': button_text, 'link': button_link }) Send a new message and trigger slaask.recieved_message event

from this attribute is optional. By default, we will use the widget "support displaying name"
_slaask.surveys() Send all your surveys to your visitor or user
_slaask.survey(survey_id) Send a single survey to your visitor or user

survey_id is your survey ID (in the order column), for example: _slaask.survey(2)
_slaask.identify(display_name, more) More information can be found here
_slaask.updateVisitorInfos(infos, callback) This function allows you to programmatically update your visitors' or users' on-page runtime. The infos parameter should be an Object containing one or both of the following keys: user_infos and current_page. The user_infos key is also used on the _slaask.identify function and should be an Object as well.
_slaask.sendMessageToSlack(message, callback) Send any message to a visitor's or user's dedicated Slack channel.
_slaask.auth(unique_identifier) Allows you to dynamically load the full widget for any of your users.

Ex.
A user is logged in on your Single Page App via their account ID 1234. They log out, and another user logs in via their own account ID 5678 on the same computer. You simply have to call _slaask.auth('a-unique-token-for-user-5678') and the widget will be fully reloaded with user 5678's information and conversation history.
