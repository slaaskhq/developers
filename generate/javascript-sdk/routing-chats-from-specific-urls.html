Routing chats from specific URLs

At Slaask, you can assign chats to specific people on your team—not only based on topics (to learn more about routing chats using topics: Understanding automatic chat dispatching, but also based on specific incoming URLs.

In other words, you can configure Slaask in such a way that only certain team members on your team will receive the requests from your leads/customers who come to your website via a specific URL.

To use this feature, here is the code your developers will need to make it happen using jQuery:

var dispatch = [
  { referrer:'referrer1.com', group_id:1 },
  { referrer:'referrer2.com', group_id:2 },
  { referrer:'referrer3.com', group_id:3 }
]

var results  = $.grep(dispatch, function(e){ return e.referrer==document.referrer; });
if(results.length==0) {
  _slaask.init("xxxxxxxxxxxxxxxxxxxxxxxxxxxx");
} else{
  _slaask.init("xxxxxxxxxxxxxxxxxxxxxxxxxxxx", { user_group_id: results[0].group_id });
}

Some useful information you should know :

  • Dispatch is an array of objects
    • The objects are composed by:
      • referrer: The referrer you want.
      • group_id: The ID of the dispatching group you want to attribute to the previous referrer (you can find those ID from your Slaask account -> Personalization -> Navigation -> Automatic chat dispatching -> The group ID is the number behind “#” in the brackets right after the name of your group.)
