Making Slaask work with Single-Page Application (SPA)

Here is a way to handle visitor's current page updates:

There is a function called updateVisitorInfos. This function, available from the _slaask global object, will force an update of the visitor information with an API call.


_slaask.updateVisitorInfos();
or 
_slaask.updateVisitorInfos({}, function(data) { 
  console.log(data);
});

With the first argument, you can override the user_infos object (_slaask.user_infos) and/or the visitorâ€™s current page (by default, window.location.href).

The second argument is a simple callback function in case you want to handle our API response.


Both arguments are optional.
